Release 8.0.0 25th December 2025
------------------------------------------------------------------

Rebase of CxPilot onto ArduPilot 4.6.3. From the operator's perspective, there
are no major functional changes to be aware of. This release fixes some
edge-case bugs, adds some new options we may enable in the future.

All changes listed in this release note will be technical changes.

### Volanti Changes

- Pusher ESC CPN no longer sends empty ESCX messages
- ELRS receiver to match Ottano
  - Set to Serial 4, which was previously used as the MAVLink pins on the RFD900
    in engineering aircraft, but will no longer be supported
  - Aircraft without RC receivers will be unaffected by this setting

### ArduPilot 4.6 Release Notes

This section is a small subset of the release notes from ArduPilot 4.6, with
only the most relevant changes to Carbonix listed. In addition, the PR number is
included to link to more discussion/code.

#### Minor Fixes and Enhancements

- Airspeed sensor health check fixed when using multiple sensors and AHRS
  affinity (25272)
- Arming check for EKF3 velocity innovations (29434)
- EK3 uses filtered velocity corrections for IMU position (26808)
- EKF3 copes better with GPS jamming (24899)
- CRSF telemetry fixes (28478 and 29931)
- Harmonic notch is active in forward flight on quadplanes (25442)
- FBWB/CRUISE climb/sink rate limited by TECS limits (26856)
- Fence re-enable after fence breach fixed (29730)
- Fixed rudder output in VTOL modes (29494)
- Improved QAssist and VTOL mode recovery from bad attitudes (29498)
- Loiter breakout improved to better handle destinations inside loiter circle
  (26102)
- Prevent learning bad `ARSP_RATIO` values when not flying (29455)
- QuadPlane assistance check enhancements (26318)
- QuadPlane fix for QLand getting stuck in pilot repositioning (28568)
- QuadPlane get smoother takeoff by input shaping target accel and velocity
  (26573)
- Terrain guided target intermediate alt handling fixed (29685)
- "open directory failed" false warning when scripts in ROMFS fixed (26312)
- Scripting gets memory handling improvements (26517)
- Various scripting performance and memory usage optimizations (27630, 27426,
  possibly others too)
- MAVFTP respects TX buffer flow control to improve FTP on low bandwidth links
  (26098)
- MCU min/max voltage logging fix (29417)
- Added option to disable IMU check with ICE running (30836)

#### New Options (not yet used)

- CAN frame logging added to ease support (28779)
- `DO_RETURN_PATH_START` mission command support (28574)

Release 7.3.0 26th November 2025
------------------------------------------------------------------

This release introduces the automatic engine-out failsafe lua script, as well as
some minor changes.

### Fleetwide Changes

- Built-in Test (BIT): the "RPM Drop" warning will trigger each individual time
  that the ESC attempts to spool up and fails to do so
- QAssist angle error threshold increased from 8 degrees to 15 degrees

### Ottano Changes

- Automatic engine-out failsafe lua script
  - This reconfigures the autopilot for glide-only flight when the engine dies,
    and makes a best-effort attempt to land at a designated alternate recovery
    site
- Built-in Test (BIT): reworded "CHT1 cold", "EGT2 cold", etc. warnings to "CHT
  difference" and "EGT difference" respectively
- Increased the EGT difference warning threshold from 50C to 75C
- Avionics battery critical voltage reduced from 22V/10s to 21.5V/60s
- PMU failure, falling back to avionics/backup battery, now triggers an RTL
- Max settable airspeed reduced from 36m/s to 33m/s
- QAssist speed decreased from 23m/s to 22m/s

### SITL Changes

- Fixed a bug where SITL was always autoconnecting to the trailer GCS like a
  real aircraft

### Technical

- Completely restructured the codebase into three distinct repositories. This
  will make it much easier going forward to keep up to date with ArduPilot
  mainline changes:
  - cxpilot: main integration and workflow host
  - cxpilot-core: ArduPilot with minimal Carbonix patches and backports
  - cxpilot-config: parameter and lua script management
- Log CircuitStatus and Temperature messages from MKS servos

Release 7.2.0 1st July 2025
------------------------------------------------------------------

This is the first release for the Octo Ottano. This contains many small
improvements to Ottano as a whole.

### Fleetwide Changes

- QAssist is significantly more sensitive
  - QAssist will trigger above 43 degrees roll angle (previously 65 degrees)
  - QAssist will trigger immediately for either angle error or low speed
    (previously a 0.5s delay)
  - This may lead to increased occurrence, particularly in high-wind/gusty
    conditions
- Bugfix: `LOITER_TURNS` with 0 turns no longer get randomly skipped
  - The aircraft must reach the zero-turn loiter
  - Additionally, if necessary, will wait until it is pointed at the next
    waypoint before exiting
- Built-in Test (BIT): "RPM Drop" response time reduced from 3 seconds to 1
  second
- Transition/QAssist now turns more gracefully
  - Previously the VTOL controller fought against coordinated turns
- Cruise: mid-stick on throttle now commands whatever cruise speed is set in the
  action panel
  - Previously, mid-stick commanded exactly halfway between min and max airspeed
- RGB LEDs on CAN GPSs now flash the correct color codes

### Ottano Changes

- Added Ottano AC_3 configuration (i.e., the Octo Ottano). Key differences with
  the older aircraft:
  - 8 VTOL motors
  - Higher speeds due to high MTOW:
    - Cruise speed: 27m/s
    - Min speed: 25m/s
    - Max speed: 36m/s
    - QAssist speed: 23m/s
  - Slower max climb rate of 2m/s due to the high MTOW
- Calibrated fuel burn to report accurately (100g or 1%, whichever is larger)
- Removed engine-start switch from controller configuration (engine is
  controlled through Planner)
- Added differential aileron
  - Downward deflection reduced to 30% of what they did previously
  - Upward deflection unchanged
- Built-in Test (BIT): Engine prearm checks and in-flight warnings:
  - CHT below 100C
  - CHT above 280C
  - EGT above 720C
  - CHT difference of 50C
  - EGT difference of 50C
  - Prearm check that greater-than 6700 RPM seen during engine run-up
- Engine warmup sequence is now automated
  - Must still be done in manual mode, but no other intervention is required
- Engine no longer throttles back partially during transition to forward flight
- Improved pusher-motor assistance in hover
- Increased weathervaning aggressiveness

### Technical

- Fixed throttle linearization fit for Ottano
- L1 nav controller improvements on Ottano
- Tighter fixed-wing tune on Ottano
- PPP networking stability fixes
- CRSF telemetry enabled in build
- Added DroneCAN option to ignore ESC offset on receive
- Fixed race condition causing random nils from lua ESC RPM getters
- SITL: fixed bug where scripting-backend battery monitors could cause a
  hardfault

Release 7.1.0 12th February 2025
------------------------------------------------------------------

This release is mostly a refactor and expansion of our warning/prearm
(Built-in-Test) lua script.

### Fleetwide Changes

- Moved wind speed warning to GCS. Disabled aircraft-generated wind speed
  warnings
- Stick mixing is disabled. In auto modes, the sticks do not do anything, with
  two exceptions:
  - The stick can still be used to yaw in VTOL auto
  - The sticks can still be used to reposition a VTOL landing

### Ottano Changes

- Increased Ottano's engine start throttle to 15%
- Safety is now engaged on boot. This was a regression from CxPilot-5.2.0

### Built-in-Test (`CX_BIT`) Changes

- Added a prearm check for sat count
  - Minimum 18 satellites
  - Maximum difference of 8 satellites between GPS1 and GPS2
- Fixed false "RPM Drop" alerts on a second takeoff
- Removed unnecessary "RPM nil" and "Servo Out nil" messages
- Removed unnecessary alerts prior to disengaging the safety switch, and a few
  seconds after
  - Wait at least 5 seconds between disengaging the safety switch and arming

### Known Issues

- "Loiter Turns" commands with 0 turns are sometimes skipped before checking
  they have been reached
- Some Ottanos under-report fuel volume from the EFI. These vary wildly between
  engines and will need calibration per aircraft

### Technical

- Fixed notch filter Q factor calculation. Additional protection against
  notch-filter issues
- Added ESC telemetry simulation
- Added Engine simulation
- Added Ottano headless simulator
- Removed `LOG_FILE_DSRMROT`. This feature cannot be used alongside EKF replay
  logging
- Fixed git hash for PR CI artifacts
- Fixed log drops causing early boot messages to be lost
- Fixed race condition in DroneCAN debug logging. This likely fixed the rare
  PIQC crash dump issue
- Disabled DroneCAN debug logging. This does not actually do anything useful for
  us anyway
- Assigned one AP servo index to each physical CAN servo. Solves the servo
  telemetry log clashes
- Fixed cx_led_driver sending its startup message as EMERGENCY (i.e., no longer
  shows on HUD)
- Added SITL safety switch engaged at boot like the real aircraft
- Added Carbonix SITL autotests. Currently only tests `CX_BIT`. More tests will
  be added later
- Improved CI packaging/uploading

Release 7.0.0 29th November 2024
------------------------------------------------------------------

This release includes moving all CxPilot changes on top of ArduPilot 4.5.

### Fleetwide Changes

- Prearm check to ensure all LUA scripts are loaded and running
- Improved back transition
- `JUMP_TAG` mission item support
- Switching to auto takeoff after manual takeoff no longer adds extra altitude
- Wind speed warning reduced to 10m/s (from 12m/s)
- Roll limit no longer confined to 7 degrees during transition
- FBWB/CRUISE fix massive undershoot issue when switching from max descent to
  max climb
- A whole bunch of bug fixes

### Ottano Changes

- Fixed-wing tune improvements. Altitude should be more stable
- Battery assignments standardized
  - Battery 1: VTOL batteries combined (reports lowest voltage)
  - Battery 2: Avionics battery
  - Battery 3: Fuel level sensor
- VTOL battery failsafe removed. Until battery failsafes can take more
  intelligent automatic action between fixed-wing and VTOL modes, the failsafe
  is dangerous. This is slated for CxPilot-7.1 (SW-466)

### Known Issues

- Unnecessary alerts from `CX_BIT`:
  - Before disengaging safety switch, nuisance alerts about missing ESC
    telemetry
  - Flying twice without rebooting causes an false positive alert about RPM drop
    on the second takeoff. Aircraft should be power cycled between flights
  - Low-priority message on the messages tab "RPM nil", will be removed in the
    next release and can still be safely ignored unless it comes up on the HUD

### Technical

- Improved support for new camera payloads and gimbals
- GPS raw logging improvements for PPK
- Added support for precision landing of quadplanes
- Added TECS option for rapid descent in fixed wing aircraft
- `AHRS_OPTIONS` supports disabling fallback to DCM
- Quadplane use of forward throttle in VTOL modes improved
- VTOL angle controller gets feed-forward scaling (see `Q_OPTIONS`)
- Arming check of compass vs world magnetic model to detect metal in ground (see
  `ARMING_MAGTHRESH`)
- `FLTMODE_GCSBLOCK` blocks GCS from changing vehicle to specified modes
- CAN port can support a second CAN protocol on the same bus (2nd protocol must
  be 11 bit, see `CAN_Dn_PROTOCOL2`)
- Ethernet/Networking support for UDP and TCP server and client (see
  `NET_ENABLED`) and PPP (see `SERIALx_PROTOCOL`)
- Firmware flashing from SD card
- Rockblock satellite modem support
- Missions can be stored on SD card (see `BRD_SD_MISSION`)
- New Notch filter features
- CubeOrangePlus supports IMU high resolution sampling (works with ICM42688,
  ICM42652, ICM42670, ICM45686 IMUs)
- Bunch of Logging enhancements
- Scripting enhancements
- CI Enhancements
  - Parameter defaults are sanity checked for simple errors like naming or
    invalid values
- ELF file artifacts now contain debug symbols
- Removed the ability to send kill commands (for testing) to the autopilot
- Added CubeOrangePlus to Carbonix-supported autopilot
- Battery sum monitors can report minimum voltage instead of average
- A7R parameter patch no longer reconfigures GPS to ellipsoid altitude
- Fixed bug which prevented full crash dump from being saved  

### Known Technical Issues

- Rare crash dumps still occurring during PIQC (has been happening since
  CxPilot-4)
- Arming again after disarming does not create a new log file
- Log messages early in boot get dropped

### Parameter changes since beta4

- Ottano fixed-wing tuning improvements
- Ottano set max engine retry parameter to 5
